## Muhammad Awad  
**Senior Member of Technical Staff @ AMD Research**

I design and lead system-level software for next-generation GPU and AI systems.

My work focuses on building programming models, runtime systems, and tooling that make large-scale GPU workloads—especially modern AI/LLM systems—easier to develop, analyze, and optimize.

**Focus areas:**  
distributed systems × GPU programming × AI workloads × performance engineering

---

## Key Projects

🔁 **[Iris](https://github.com/ROCm/iris)**  
Triton-based multi-GPU programming framework for scalable distributed GPU execution.  
Enables fusion-driven execution and communication–computation overlap for high-performance AI workloads.

🧠 **[IntelliKit](https://github.com/AMDResearch/intellikit)**  
LLM-native GPU profiling and analysis toolkit.  
Transforms low-level hardware counters, traces, and assembly into structured, actionable insights.

⚡ **[IntelliPerf](https://github.com/AMDResearch/intelliperf)**  
Autonomous GPU performance engineering system.  
Profiles, diagnoses, and optimizes kernels end-to-end using AI-driven workflows.

⚙️ **[IRON (MLIR-AIE)](https://github.com/Xilinx/mlir-aie)**  
Contributor to a low-level development stack for AMD Ryzen™ AI NPUs, including Python APIs and MLIR-based compilation flows.

## System Perspective

Together, these projects form an end-to-end stack for AI-driven GPU performance engineering:

- **Iris** → programming model  
- **IntelliKit** → observability + tooling  
- **IntelliPerf** → automation + optimization  

Bridging the gap between low-level GPU execution and high-level AI systems.
